Feature Plan

Monthly Log + Budget — สิ่งที่ต้องปรับ

สรุปความเข้าใจก่อน code — confirm แล้วค่อยลงมือ

ที่เข้าใจจากที่บอก

เข้าใจแล้ว

1. Income ดึงจาก Profile อัตโนมัติ

ตอนเปิด modal กรอก Monthly Log — ช่อง รายได้ ควร pre-fill ด้วย annual_income ÷ 12 จาก Profile ให้เลย ไม่ต้องพิมพ์ซ้ำทุกเดือน แก้ได้ถ้าอยากใส่ต่างออกไป

เข้าใจแล้ว

2. Monthly modal ดึง Budget มาคำนวณรายจ่าย

เมื่อกดเปิดเดือน — modal จะแสดง ทุก sub-category จาก Budget ให้กรอก actual รายหมวด แล้ว รวม auto เป็นรายจ่ายรวม ไม่ต้องคิดเองและตรง category เลย

เข้าใจแล้ว

3. Budget — เพิ่ม Sub-category ได้ไม่อั้น

ปัจจุบัน add ได้แต่ต้องเปิด modal แยก — ปรับให้ inline ในตารางทุกหมวด กดปุ่ม + แล้วพิมพ์ชื่อ sub-category ใหม่ได้เลยไม่ต้องเปิด popup

เข้าใจแล้ว

4. Flow: Budget ก่อน → Monthly Log ทีหลัง

ต้องตั้ง Budget ให้เสร็จก่อน แล้วค่อยมากรอก Monthly Log — เพราะ Monthly จะดึง category จาก Budget มาแสดง ถ้า Budget ยังว่างอยู่ Monthly จะไม่มี category ให้กรอก

Flow ที่จะเปลี่ยน

เดิม Monthly modal → พิมพ์ income เอง → พิมพ์ expenses รวม → บันทึก
ใหม่ Budget ตั้งค่าก่อน → Monthly modal → income = profile ÷ 12 (auto) → กรอก actual ทีละหมวด → expenses รวมอัตโนมัติ → บันทึก

สรุปสิ่งที่ต้อง code

1
src/routes/monthly.py — Pre-fill income จาก Profile ส่ง profile.annual_income / 12 ไปใน context ของ monthly template ให้ value ใน input income = ค่านี้ถ้ายังไม่เคยกรอก
2
src/routes/monthly.py — ดึง Budget categories ส่งไป template query budget_categories WHERE year=? group by category ส่งไปด้วย เพื่อให้ modal แสดงทุก sub-category พร้อม budget amount
3
src/templates/monthly.html — Modal ใหม่ (budget-linked)
  • income = monthly_income จาก profile (แก้ได้)
  • แสดง sub-category ทุกหมวดพร้อม input actual แต่ละรายการ
  • JS รวม actual ทุก input เป็น total expenses แบบ real-time
  • savings rate = auto จาก income - expenses
4
src/routes/monthly.py (POST) — รับ actual แต่ละ category รับ form fields actual_[id] ทุกตัว → sum เป็น expenses → upsert monthly_log → optional: upsert กลับไปที่ budget_categories.actual_ytd ด้วย
5
src/templates/budget.html — Inline add sub-category แต่ละ category section มี row ว่าง + ปุ่ม + อยู่ท้ายตาราง กรอกชื่อ sub-category + budget/เดือน แล้วกด เพิ่ม → POST inline ไม่ต้องเปิด modal

ยืนยัน 2 จุดก่อน code

1 เมื่อกรอก actual ในแต่ละ category แล้ว อยากให้อัปเดต Budget actual_ytd ไปด้วยเลยไหม? หรือให้ Monthly Log กับ Budget แยกกัน (กรอก actual ใน Budget แยก)
2 ถ้า Budget ของปีนั้นยังไม่มี category เลย — modal Monthly จะแสดงแค่ income + net worth field แบบเดิม หรืออยากให้ redirect ไปตั้ง Budget ก่อนไม่ให้กรอก Monthly ได้?
