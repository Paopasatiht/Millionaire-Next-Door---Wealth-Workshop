Concept Discussion

ข้อมูลอยู่ที่เครื่องคุณเท่านั้น

เลือก storage strategy สำหรับ MND Wealth Workshop — ก่อนลงมือเขียน code

🔒

หลักการที่ไม่เปลี่ยน: ข้อมูลการเงินส่วนตัวอยู่บนเครื่องผู้ใช้เท่านั้น ไม่มีการส่งออก ไม่มี server รับ ไม่มี cloud sync — GitHub ได้รับแค่ไฟล์ project ข้อมูลจริงอยู่ใน data/ ที่ถูก .gitignore ไว้

Option A

IndexedDB — ฝังใน Browser

ดับเบิ้ลคลิก HTML ได้เลย ข้อมูลอยู่ใน browser storage

เปิด HTML → Browser กรอกข้อมูล → IndexedDB ใน Browser → ข้อมูลอยู่ใน browser นั้น

ข้อดี

  • ดับเบิ้ลคลิกเปิดได้ทันที ไม่ต้องรัน server
  • ทำงานได้ทุก browser
  • เขียน code ง่ายที่สุด

ข้อเสีย

  • ข้อมูลผูกกับ browser — ล้าง cache = หายหมด
  • เปลี่ยน browser หรือเครื่องใหม่ ต้อง export ก่อน
  • ไม่มีไฟล์ .db จริงๆ ให้จัดการหรือ backup
⭐ แนะนำ
Option B

SQLite WASM — ไฟล์ .db บนดิสก์จริง

SQL จริง, ไฟล์จริงบนเครื่อง, เปิด HTML ได้เลยไม่ต้องรัน server

เปิด HTML → SQLite ทำงานใน Browser (WASM) → บันทึก → mnd-data.db บนดิสก์ → เปิดครั้งต่อไปโหลดไฟล์เดิม

ข้อดี

  • SQL จริงๆ — query, join, aggregate ได้เต็มรูปแบบ
  • ไฟล์ .db จับต้องได้ — copy, backup, ย้ายเครื่องได้
  • ดับเบิ้ลคลิกเปิดได้ ไม่ต้องรัน server
  • ข้อมูลไม่หายแม้ล้าง browser cache
  • ต่อยอดเปิดด้วย DB Browser for SQLite ได้

ข้อเสีย

  • รองรับเต็มที่บน Chrome / Edge — Firefox มีข้อจำกัด
  • ต้อง load WASM library (~1MB) ครั้งแรก
  • ผู้ใช้ต้อง "อนุญาต" ให้ browser เข้าถึงโฟลเดอร์ครั้งแรก
Option C

Python Flask + SQLite — Local Server

รัน python app.py ก่อน แล้วเปิด localhost:5000 ในบราวเซอร์

python app.py → localhost:5000 → Flask รับ request → mnd-data.db บนดิสก์

ข้อดี

  • SQL เต็มรูปแบบ ไม่มีข้อจำกัด browser
  • ไฟล์ .db บนดิสก์ชัดเจน
  • รองรับ Python ที่มีอยู่แล้วในเครื่อง
  • ขยายต่อ (export Excel, report) ง่ายมาก

ข้อเสีย

  • ต้องรัน server ทุกครั้งก่อนใช้งาน
  • ต้องติดตั้ง Python + Flask
  • เปิดแบบ double-click ไม่ได้

เปรียบเทียบ

เกณฑ์ A — IndexedDB B — SQLite WASM ⭐ C — Flask + SQLite
เปิดด้วย double-click ✓ ได้เลย ✓ ได้เลย ✗ ต้องรัน server ก่อน
ข้อมูลเป็นไฟล์จริงบนดิสก์ ✗ อยู่ใน browser ✓ .db file บนดิสก์ ✓ .db file บนดิสก์
ข้อมูลปลอดภัยถ้าล้าง cache ✗ หายหมด ✓ ไฟล์อยู่บนดิสก์ ✓ ไฟล์อยู่บนดิสก์
SQL เต็มรูปแบบ ✗ Object store ✓ Full SQL ✓ Full SQL
ย้ายเครื่อง / backup △ Export ก่อน ✓ Copy .db ไฟล์ ✓ Copy .db ไฟล์
ต้องติดตั้งอะไรเพิ่ม ✓ ไม่ต้อง ✓ ไม่ต้อง (Chrome/Edge) ✗ Python + Flask
ความซับซ้อนในการ implement ต่ำ กลาง กลาง

ก่อนตัดสินใจ — ช่วยตอบ 3 ข้อนี้

1
Browser หลักที่ใช้คืออะไร? Option B ทำงานได้เต็มที่บน Chrome / Edge — ถ้าใช้ Firefox เป็นหลักควรเลือก A หรือ C แทน
2
ใช้คนเดียวบนเครื่องนี้เครื่องเดียว หรืออยากย้ายข้อมูลระหว่างเครื่อง? ถ้าย้ายเครื่อง — Option B และ C ดีกว่าชัดเจน เพราะ copy ไฟล์ .db ได้เลย
3
โอเคกับการกด "อนุญาต" ให้ browser เข้าถึงโฟลเดอร์ data/ ครั้งแรกไหม? Option B ต้องให้ permission ครั้งแรก (เหมือนที่เว็บขอเข้า mic/camera) — ครั้งเดียวพอ ครั้งต่อไปจำเองอัตโนมัติ
